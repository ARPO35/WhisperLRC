# 低置信度人工审核 UI 方案

## 概述
在 ASR+翻译流程后新增本地 Web UI，用于低置信度人工复核。  
目标：让审核人员高效修正高风险句子，并导出纠正后的双语 LRC：
- `<basename>.final.ja.lrc`
- `<basename>.final.zh.lrc`

本阶段是产品/工程层面的高层方案，不是逐行实现规范。

## 范围
### 包含内容
- 本地单用户 Web 应用
- FastAPI + React 架构
- 按“低置信度 + 规则命中”构建句子队列
- 以句子为主的审核流程，支持可选的词级上下文
- 基础音频控制：播放/暂停、跳转句子、按时间戳拖动
- 使用 JSON 旁路文件持久化审核状态与编辑内容
- 导出最终双语 LRC 文件

### 不包含内容
- 多用户认证与权限
- 云端协作
- DAW 级波形编辑

## 目标与成功标准
### 目标
1. 自动收集高风险句子并进入审核队列。
2. 支持人工高效修正日文与中文文本。
3. 保留原始时间轴并重新生成最终 LRC 输出。

### 成功标准
1. 审核工作量低于完整人工重听。
2. 修正后的句子以稳定时间戳导出。
3. 失败项与部分导出结果可追踪、可恢复。

## 输入与输出
### 输入
- ASR 句子级结果（时间戳、置信度、可选词级时间）
- 初始文件：
  - `<basename>.ja.lrc`
  - `<basename>.zh.lrc`

### 输出
- `<basename>.final.ja.lrc`
- `<basename>.final.zh.lrc`
- 可选导出日志（用于记录警告/错误）

## 高层工作流
1. 批量 ASR+翻译完成后，生成候选审核任务。
2. 队列构建器筛选满足以下条件的句子：
   - 置信度低于阈值
   - 命中规则（例如长句无标点、异常重复）
3. 审核人员在 Web UI 中打开任务，听音并编辑文本。
4. 保存时增量写入旁路 JSON（支持中断后恢复）。
5. 导出时合并编辑结果并输出最终双语 LRC。

## UI 信息架构
### 1) 任务列表页
- 文件级状态：`todo`, `in_review`, `done`, `partial`, `failed`
- 风险指标：低置信度数量、规则命中数量
- 支持按风险和剩余工作量排序

### 2) 审核工作区
- 左侧：句子队列（高风险优先）
- 中间：音频播放器 + 时间轴拖动
- 右侧：可编辑字段（JA/ZH）、原文、可选词级提示
- 操作：保存当前、标记通过、跳转下一条高风险句子

### 3) 导出页
- 导出最终 JA/ZH LRC
- 显示完成摘要与失败信息

## 风险规则（初版）
1. 置信度低于可配置阈值。
2. 长句缺少标点。
3. 重复/乱码模式。
4. JA-ZH 长度比例异常（翻译质量提示）。

## 错误处理与降级策略
1. 句子保存失败：保留本地表单状态并允许重试。
2. 导出失败：保持旁路文件不变，并允许重新导出。
3. 翻译缺失：允许仅修正 JA；导出时标记 ZH 警告。
4. 音频加载失败：任务仍可编辑，并显示明确警告。

## 验收测试场景
1. 低风险文件可被快速审核通过并导出。
2. 高风险文件支持逐句修正与音频定位。
3. 应用重启后可继续审核（状态已持久化到旁路文件）。
4. 导出的最终 LRC 保留原句子时间轴。
5. 局部失败在任务页与导出日志中可见。

## 里程碑
1. M1：旁路结构 + 任务列表 + 基础播放器
2. M2：审核工作区 + 增量保存/恢复
3. M3：最终 LRC 导出 + 报告
4. M4：规则调优与体验打磨

## 锁定默认项
1. 本地单用户部署
2. FastAPI + React 技术栈
3. 队列策略：低置信度 + 规则命中
4. 审核粒度：以句子为主，辅以词级上下文
5. 输出策略：覆盖式最终产物（`final.ja.lrc`, `final.zh.lrc`）
